## 🧠 THEME OF THE DAY

**Theme:** Software Dev & AI Building

**What moved today**
Claude Cowork launched and is being favorably compared to OpenClaw. It was built in ~10 days by a small team using Claude Code instances orchestrating each other — the tool wrote itself. Felix Rieseberg (Slack desktop, Electron core maintainer) shipped it using Electron with deliberate sandboxing choices. Separately, a real prompt injection attack chain in Snowflake's Cortex Agent was disclosed and patched: a malicious README caused the agent to execute a remote shell command because `cat` was on an approved command list without protection against process substitution. Simon Willison's takeaway: allow-lists on agent commands are inherently unreliable; deterministic sandboxes outside the agent layer are the right answer. On the local inference front, Qwen3.5-397B-A17B is now running at 5.5+ tokens/second on a 48GB MacBook M3 Max by streaming MoE expert weights from SSD — a technique derived from Apple's 2023 "LLM in a Flash" paper, with Claude Code used to implement the research auto-experimentally.

**Why it matters**
Three things are converging fast: (1) agentic desktop tools are shipping and self-replicating — Claude Cowork wrote itself, which means the barrier to building agentic infra is collapsing; (2) the security surface of agents touching external content (repos, READMEs, URLs) is now empirically dangerous, not theoretically; (3) local inference for very large models is becoming practical on prosumer hardware. For Julien's 6-month horizon scan, these are not incremental updates — they are category shifts. The "AI building AI" loop is live in production. The blog pipeline, the Staffing Calendar System, and any future agentic tooling Julien ships all sit inside this threat/opportunity envelope.

**What to do or watch**
1. **Security audit posture:** Any skill or agent in Julien's stack that reads external content (URLs, GitHub repos, user-submitted text) should be treated as a prompt injection surface. The Snowflake case is the template — no allow-list is safe. If NightCrawler or any pipeline skill touches external sources, verify there is a layer of human approval or deterministic sandboxing between the agent and execution.
2. **Claude Cowork as a reference architecture:** The "team of humans orchestrating Claude Code instances" pattern used to build Claude Cowork is directly applicable to Julien's blog pipeline and business tooling. Worth reading the full Latent Space pod for the design thinking — especially the sandboxing and Electron choices.
3. **Watch:** Whether Mistral Forge (enterprise fine-tuning on proprietary data) gains traction as an alternative to prompt engineering for institutional knowledge — relevant to any future Chez Julien AI tooling.
