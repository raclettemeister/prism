## 🏗️ BUILD WATCH

**The "AI-rewrite-to-relicense" legal trap is live.** The `chardet` v7.0.0 case — where maintainers used Claude Code to rewrite LGPL code and re-release under MIT — has surfaced a genuine legal paradox: if AI-generated code lacks copyright standing (per the U.S. Supreme Court's March 2 non-ruling), the relicensed output may simultaneously be (a) a derivative work still bound by LGPL, and (b) uncopyrightable, potentially public domain. A clean-room rewrite requires two isolated teams — Team A documents the spec, Team B writes from scratch without seeing the original. By prompting an AI with the original LGPL code, that wall is bypassed. For Julien: David Toolkit is AGPL. Any contributor who tries to "rewrite" a dependency using an AI tool trained on the original could inadvertently create a licensing landmine. Worth adding a note to the contributor docs.

**`406.fail` — open-source maintainer backlash against AI-generated PRs is becoming infrastructure.** A standard protocol now exists to formally reject AI-generated contributions and instruct the submitting agent to halt. The spec defines a standard for handling and discarding low-effort, machine-generated contributions to source code repositories, issue trackers, and community forums. This is a cultural signal: the open-source community is bifurcating between projects that welcome AI-assisted contributions and those that actively wall them out. Grassroot Hoppers sits on the welcoming side — worth making that posture explicit in the contributor docs.

**SurvivalIndex.org** — a new platform rating open-source projects on their likelihood of surviving the AI era, using six criteria including "Agent Friction" and "Insight Compression." The core insight: not all software will be re-synthesized by AI — software with high survival scores represents crystallized knowledge that would be inefficient to recreate from scratch. Directionally useful for Julien's reconnaissance mission: if you're choosing which infrastructure to build on, build on S-tier and A-tier projects (Git: 9.6, PostgreSQL: 9.1). Avoid building on anything that AI will simply replace.
